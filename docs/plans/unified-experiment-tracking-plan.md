## Summary

Unify the 4 disconnected experiment tracking systems (ML trainer, SFT/KTO trainers, Evaluator, cloud training) into a single centralized tracking layer. The approach uses a common `RunRecord` schema stored in an append-only JSONL registry file, with thin adapters that bridge each system's existing lineage output into the unified index. Existing lineage files remain the source of truth for detailed run data; the registry is a lightweight, queryable index with cross-run linkage. Auto-registration happens inside the tracker's `start_run()` context manager exit, so callers get tracking for free.

#### Design Approach

**Architecture Principles:**
1. **Registry Pattern** — Append-only JSONL index file that all systems write to
2. **Adapter Pattern** — Thin adapters per trainer bridge existing lineage → RunRecord
3. **Schema Inheritance** — Common RunRecord base; type-specific details stay in existing lineage files
4. **Read/Write Separation** — ExperimentTracker handles writes; RunRegistry handles queries
5. **Auto-Registration** — `start_run()` context manager auto-registers on successful exit

#### Interface Contracts

**1. RunRecord Schema**
```python
@dataclass
class RunRecord:
    """Common fields across ALL run types. Stored in registry.jsonl."""
    run_id: str              # UUID4
    run_type: str            # "sft" | "kto" | "ml" | "evaluation" | "cloud_sft" | "cloud_kto"
    name: str                # Human-readable run name
    timestamp: str           # ISO 8601 UTC
    status: str              # "completed" | "failed" | "running"
    output_dir: str          # Absolute path to run artifacts
    parent_run_id: str | None  # Links eval → training run
    tags: dict[str, str]     # Flexible metadata (method, provider, etc.)
    schema_version: int      # For forward compatibility (start at 1)

    # Common optional fields (not all types have all)
    model_name: str | None
    dataset_source: str | None
    primary_metric: float | None
    primary_metric_name: str | None
    hardware: str | None     # e.g., "RTX 3090", "A100", "CPU"
```

**2. RunRegistry (query.py)**
```python
class RunRegistry:
    def __init__(self, registry_path: Path): ...
    def register_run(self, record: RunRecord) -> str: ...
    def link_runs(self, child_run_id: str, parent_run_id: str, relationship: str) -> None: ...
    def find_runs(self, filters: RunFilter | None = None) -> list[RunRecord]: ...
    def get_run(self, run_id: str) -> RunRecord | None: ...
    def get_linked_runs(self, run_id: str, relationship: str | None = None) -> list[RunRecord]: ...
```

**3. RunFilter (query.py)**
```python
@dataclass
class RunFilter:
    run_type: str | list[str] | None = None
    status: str | None = None
    model_name: str | None = None
    since: str | None = None       # ISO 8601
    until: str | None = None       # ISO 8601
    tags: dict[str, str] | None = None
```

**4. ExperimentTracker ABC Extension**
```python
class ExperimentTracker(ABC):
    # Existing abstract methods unchanged:
    # set_experiment, start_run, log_params, log_metrics, log_artifact

    # New convenience method (non-abstract, default no-op):
    def log_metadata(self, key: str, value: Any) -> None:
        """Log arbitrary metadata. Default no-op."""
        pass
```

**5. Auto-Registration Flow**
```python
# Inside LocalTracker.start_run() context manager __exit__:
# On successful completion, auto-register RunRecord to RunRegistry
# On exception, skip registration (run failed)
```

**6. Registry File Format (registry.jsonl)**
```jsonl
{"run_id":"uuid","run_type":"sft","name":"SFT run 2026-03-19","timestamp":"2026-03-19T12:00:00Z","status":"completed","output_dir":"/path/to/run","parent_run_id":null,"tags":{"method":"sft","provider":"local"},"schema_version":1,"model_name":"Qwen3-4B","dataset_source":"Datasets/tools.jsonl","primary_metric":1.286,"primary_metric_name":"final_loss","hardware":"RTX 3090"}
{"run_id":"uuid","run_type":"evaluation","name":"Eval of SFT run","timestamp":"2026-03-19T13:00:00Z","status":"completed","output_dir":"/path/to/eval","parent_run_id":"<sft-uuid>","tags":{"suite":"full_coverage"},"schema_version":1,"model_name":"Qwen3-4B","dataset_source":null,"primary_metric":0.45,"primary_metric_name":"pass_rate","hardware":null}
```

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Registry file corruption (partial write) | Low | Medium | Append-only JSONL — partial corruption loses only last entry. Write to temp file + atomic rename |
| Existing trainers break | Very Low | High | Post-training hook is additive, try/except wrapped. If registration fails, training still completes |
| Run ID collisions | Negligible | Low | UUID4 — effectively zero collision probability |
| Schema evolution breaks old records | Low | Medium | `schema_version` field + reader ignores unknown fields (forward-compatible) |
| Pre-commit hook false positives | Medium | Low | Avoid `print(...token...)` patterns in new code. Use logging module instead |
| ABC extension breaks ML trainer | Very Low | Medium | `log_metadata()` has default no-op implementation. Existing code unaffected |
| Concurrent writes from parallel training | Low | Low | JSONL append is file-system atomic for reasonable line lengths. Lock file if needed |
