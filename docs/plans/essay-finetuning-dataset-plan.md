# Implementation Plan: Essay Fine-tuning Dataset Pipeline

> Generated by `/PACT:plan-mode` on 2026-02-14
> Status: PENDING APPROVAL

## Summary

Create training data from ~117 Meditations on Alignment essays for two-stage fine-tuning: **Stage 1** (messy thoughts → structured outline) and **Stage 2** (outline → full essay with frontmatter). Uses SynthChat's existing config-driven infrastructure for LLM work. **No changes to repo code.** One disposable script in `scratch/` (gitignored) for Stage 2 assembly.

---

## Two-Stage Dataset

**Stage 1** — Thoughts to Outline (user brainstorms → model outlines):
```jsonl
{
  "conversations": [
    {"role": "user", "content": "I want to write about coding with LLMs being like playing music by ear. Thoughts: started coding a year ago, scared of syntax, Beatles couldn't read music, my sax experience, prototype addiction..."},
    {"role": "assistant", "content": "# Writing Assignment: Code Jammin'\n\n## Overview\n...\n\n## Structure\n### Opening: Personal coding journey\n..."}
  ],
  "label": true
}
```

**Stage 2** — Outline to Essay (model produces essay with frontmatter):
```jsonl
{
  "conversations": [
    {"role": "user", "content": "# Writing Assignment: Code Jammin'\n\n## Overview\n...\n\n## Structure\n..."},
    {"role": "assistant", "content": "---\ntitle: \"Code Jammin'\"\nsubtitle: \"Learning to Code Without Reading Music\"\ndate: 2025-10-04\ntags:\n  - AI\n  - coding\n---\n\n# Code Jammin'\n\nI started coding with the help of LLMs about a year ago..."}
  ],
  "label": true
}
```

## What Gets Created

### 1. SynthChat Scenario YAML: `SynthChat/scenarios/essay_outline.yaml`

**Purpose**: Stage 1 extraction. SynthChat loads each essay via `--docs` mode (`{doc_content}` template variable). LLM reads the essay and generates:
- **User turn**: Messy brainstorm/thoughts simulating what the author might have dumped before writing
- **Assistant turn**: Structured detailed outline

**Run with**:
```bash
python -m SynthChat.run generate --docs "Meditations on Alignment/" --scenarios essay_outline --per-doc 1
```

**Note**: SynthChat's `DocsLoader` doesn't recurse into subdirectories. The essays in series folders (Divine Sparks/, Constructive Rebellion/, etc.) won't be picked up automatically. Workaround options:
- Pre-flatten: disposable script copies/symlinks all essays to a flat temp directory
- Run `--docs` per subdirectory manually
- Quick enhancement to DocsLoader (but user prefers no repo code changes)

### 2. Disposable Script: `scratch/essay_dataset/pair_with_essays.py`

**Purpose**: Stage 2 assembly. Takes Stage 1 JSONL output, extracts the outlines (assistant responses), pairs each with the original cleaned essay (with frontmatter). **No LLM calls — pure data assembly.**

Steps:
1. Read Stage 1 JSONL
2. For each entry, extract the assistant response (the outline)
3. Load corresponding original essay (tracked via metadata/doc_path)
4. Clean essay: strip social media sections, Obsidian artifacts (`![[]]`, `[[]]`, callouts). **Keep YAML frontmatter.**
5. Output Stage 2 JSONL: user = outline, assistant = essay with frontmatter

### 3. SynthChat Scenario YAML: `SynthChat/scenarios/essay_synthetic.yaml`

**Purpose**: Generate ~117 novel synthetic pairs for both stages. No `--docs` input — LLM generates new topics in the author's thematic space with style guidance and few-shot examples baked into the prompts.

**Run with**:
```bash
python -m SynthChat.run generate --scenarios essay_synthetic --count 117
```

## Essay Cleanup Rules

**Strip**:
- Social media sections (YouTube descriptions, LinkedIn posts, image descriptions)
- `![[image]]` Obsidian embeds
- `[[wikilink]]` syntax → plain text
- `>[!callout]` blocks
- Research notes (`*/Research/*` directories)
- Index/map files (`_prefix`, emoji prefix files)
- Files < 500 words

**Keep**:
- YAML frontmatter (title, subtitle, description, date, tags)
- `# ` and `## ` headings
- Standard markdown formatting
- `---` horizontal rules

**Dedup**: Content hash, keep latest version only (An Emulcifier has 4 versions).

## Implementation Sequence

1. Create `SynthChat/scenarios/essay_outline.yaml` (config only)
2. Test run on 3-5 essays, inspect quality, iterate on prompts
3. Full corpus run → Stage 1 JSONL
4. Create `scratch/essay_dataset/pair_with_essays.py` (disposable)
5. Run pairing script → Stage 2 JSONL
6. Create `SynthChat/scenarios/essay_synthetic.yaml` (config only)
7. Run synthetic generation → synthetic Stage 1 + Stage 2 JSONL
8. Spot-check quality across all outputs
