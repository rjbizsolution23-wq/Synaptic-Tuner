# Implementation Plan: SFT Experiment Pipeline with Data Quality Scoring

> Generated by `/PACT:plan-mode` on 2026-03-20
> Status: PENDING APPROVAL

## Summary

Adds a multi-model SFT experiment pipeline that computes per-example loss after training, compares loss signals across multiple trained models and a shared base model, and outputs a LightGBM-ready feature matrix. An optional LLM judge step labels a sample of examples for eventual quality model training. Supports two execution modes: **local** (GPU + `.tracking/`) and **cloud** (HF Jobs end-to-end, artifacts in HF bucket). Cloud runs stay cloud — training, loss computation, comparison, and eval all run on HF Jobs; a download command optionally pulls results to local `.tracking/`.

### Architectural Principle: Cloud Stays Cloud

> If a training run was done in the cloud, all downstream steps (loss computation, eval, comparison) also run in the cloud. Artifacts live in the HF bucket. Local download is optional and one-way.

```
LOCAL mode:
  tuner.py train → compute-losses → compare-runs → judge-sample
  Artifacts: .tracking/{run_id}/, .tracking/experiments/{id}/

CLOUD mode:
  tuner.py cloud-pipeline → [HF Jobs: train + compute-losses] → bucket
  tuner.py cloud-compare  → [HF Jobs: compare-runs]           → bucket
  tuner.py download-experiment --experiment-id X              → .tracking/ (optional)
```

#### Design Approach

Post-hoc inference pipeline: after training completes, the trained model is run over each JSONL example sequentially (batch_size=1, no shuffle, `torch.no_grad()`). Completion-only masking replicates TRL's `DataCollatorForCompletionOnlyLM` by searching for the `response_template` token sequence in `input_ids` and masking everything before it. The base model runs once per experiment (not per run). A comparison script joins all per-run loss files into a stable feature matrix.

#### Key Decisions

| Decision | Options | Recommendation | Rationale |
|----------|---------|----------------|-----------|
| Masking strategy | Custom heuristic vs. replicate TRL | Replicate TRL response_template detection | Must match training exactly; mismatch biases loss values |
| compare_runs location | tuner.py subcommand vs. Tools/ script | Tools/ standalone script | Analysis tool, no GPU needed, matches existing Tools/ pattern |
| Base model losses storage | Per-run vs. per-experiment | Per-experiment (experiment dir) | Compute once, shared across all runs using same base model |
| Experiment ID format | UUID vs. timestamp vs. hash | Timestamp `exp_YYYYMMDD_HHMMSS` | Human-readable, sortable, zero collision risk for local tool |
| Judge tool | Integrated into compare_runs vs. separate | Separate `Tools/judge_sample.py` | Different dependency profile (LLM backend required); optional step |
| features.csv column naming | Run-ID-based vs. positional (`loss_run_0`) | Positional | Schema stable across different experiments; run IDs recorded in experiment.json |

#### Interface Contracts

**`LossResult` dataclass** (in `schema.py`):
```python
@dataclass
class LossResult:
    index: int                    # JSONL line index (0-based)
    loss: float                   # Mean cross-entropy on completion tokens only
    num_completion_tokens: int    # Non-masked token count
    num_total_tokens: int         # Total tokenized sequence length
    jsonl_hash: str               # First 8 chars of SHA-256 of raw JSONL line
```

**`compute_per_example_losses()` signature**:
```python
def compute_per_example_losses(
    model,                        # Trained model in eval mode
    tokenizer,                    # Tokenizer with chat template
    dataset_path: Path,           # Training JSONL
    max_seq_length: int = 2048,   # Must match training config
    completion_only: bool = True, # Must match training config
    device: str | None = None,    # Auto-detect if None
) -> list[LossResult]
```

**`experiment.json` schema**:
```json
{
  "experiment_id": "exp_YYYYMMDD_HHMMSS",
  "name": "human-readable label",
  "created_at": "ISO8601",
  "dataset_path": "relative/path/to/training.jsonl",
  "dataset_hash": "sha256:...",
  "base_model_name": "unsloth/model-name",
  "run_ids": ["uuid-1", "uuid-2", "uuid-3"],
  "base_losses_path": ".tracking/experiments/{id}/base_losses.jsonl",
  "features_csv_path": ".tracking/experiments/{id}/features.csv",
  "judge_scores_path": null,
  "status": "partial|complete"
}
```

**`features.csv` column contract (STABLE — LightGBM depends on this)**:

Tier 1 — Core (frozen after first LightGBM training):
- `index`, `loss_run_0..N`, `loss_base`, `loss_delta_run_0..N`, `loss_mean`, `loss_std`, `loss_min`, `loss_max`, `delta_mean`, `num_completion_tokens`

Tier 2 — Extended (NaN-safe, added incrementally):
- `num_turns`, `num_tool_calls`, `response_length`, `instruction_length`, `has_code`, `ifd_score`, `entropy_mean`

Tier 3 — Labels (target variable for LightGBM):
- `judge_quality` (nullable — from optional judge step)

**`.tracking/` directory layout**:
```
.tracking/
  registry.jsonl                        # existing
  links.jsonl                           # existing
  {run_id}/
    per_example_losses.jsonl            # new (per trained run)
  experiments/
    {experiment_id}/
      experiment.json
      base_losses.jsonl                 # base model losses (shared)
      features.csv                      # LightGBM-ready feature matrix
      judge_scores.jsonl                # optional LLM judge labels
```

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Masking mismatch (post-hoc ≠ training) | Medium | High — biases loss values | Extract response_template from config; include masking metadata in output; unit test boundary detection |
| features.csv schema drift | Low | Medium — breaks LightGBM models | Tier system; schema stability test asserts exact column names |
| Partial experiment (< N runs) | Medium | Low — statistical properties change | status field in experiment.json; compare_runs.py works with any N ≥ 1 |
| Dataset version mismatch across runs | Low | High — index misalignment | jsonl_hash validation in compare_runs.py; dataset_hash in experiment.json |
| OOM during post-hoc inference | Low | Medium — hook fails silently | try/except hook; explicit cache clear; batch_size=1 |
